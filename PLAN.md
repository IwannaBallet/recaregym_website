# 리케어짐 홈페이지 기획서

> 체형교정 · 재활 · PT 전문 피트니스 센터
> 목적: 대표 컨펌용 시범/데모

---

## 1. 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | 리케어짐 (ReCare GYM) |
| 목적 | 홈페이지 + 트레이너 예약 시스템 |
| 단계 | 시범/데모 (대표 컨펌 후 실서비스 전환) |
| 페이지 구성 | 단일 페이지 (Single Page) |

---

## 2. 페이지 구조

```
┌──────────────────────────────────────┐
│  HERO                                │
│  리케어짐 로고 + 한줄 소개 + 특장점  │
├──────────────────────────────────────┤
│  TRAINERS                            │
│  트레이너 15명 카드 목록              │
│  사진 / 이름 / 전문분야 / [예약하기] │
├──────────────────────────────────────┤
│  FOOTER                              │
│  주소 / 전화번호 / 운영시간           │
│  인스타그램 / 네이버 지도 링크        │
└──────────────────────────────────────┘
```

---

## 3. 예약 플로우

### 고객 (유저) 입장

```
[예약하기] 버튼 클릭
    → 모달 팝업 열림
    → Step 1: 날짜 선택
    → Step 2: 시간 선택 (1시간 단위, 이미 예약된 슬롯 비활성화)
    → Step 3: 이름 / 연락처 / 문의사항 입력
    → 제출
    → "예약 요청이 완료됐습니다. 선생님 확인 후 확정 안내드립니다." 메시지
```

### 트레이너 (선생님) 입장

```
예약 요청 들어옴
    → 트레이너 이메일로 알림 수신 (예약자 정보 포함)
    → Google Sheets 열기
    → 해당 예약 행의 [status] 열을 "대기중" → "확정" 또는 "거절" 로 변경
    → Apps Script 자동 감지
    → 고객에게 결과 이메일 자동 발송
```

## 5. Google Sheets 구조

### `trainers` 시트 — 트레이너 정보

| 컬럼 | 타입 | 예시 |
|---|---|---|
| id | text | trainer_01 |
| name | text | 홍길동 |
| specialty | text | 체형교정, 재활 |
| slot_start | text | 09:00 |
| slot_end | text | 21:00 |
| available_days | text | 월,화,수,목,금 |

| is_active | boolean | TRUE |

> 데모 단계: 전원 동일 시간대 하드코딩 → 실서비스 전환 시 개인별 설정

| trainer_email | text | trainer@recaregym.com |

| booking_time | text | 14:00 |

## 6. Apps Script 로직

### Web App — 예약 저장 (`doPost`)
1. 프론트에서 예약 데이터 POST 수신
2. `bookings` 시트에 새 행 추가 (status: "대기중")
3. 트레이너 이메일로 예약 알림 발송
4. 고객에게 "예약 요청 접수" 확인 이메일 발송

### onEdit 트리거 — 상태 변경 감지
1. `bookings` 시트의 `status` 컬럼 변경 감지
2. "확정" 으로 변경 시 → 고객에게 확정 이메일 발송
3. "거절" 으로 변경 시 → 고객에게 거절 + 재예약 안내 이메일 발송
